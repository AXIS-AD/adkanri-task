# 役割別表示とタスク管理統合の設計

## 目標

- **依頼者**（運用者）: 自分が依頼したタスクの状況だけ見れる
- **アド管理メンバー**（4人）: 全体のタスクが見える・進捗更新できる
- **task-dashboard**（localhost:3456）をアド管理用に活用

---

## 役割の定義

| 役割 | 見える範囲 | できること |
|------|-----------|-----------|
| **依頼者** | 自分が依頼したタスクのみ | 状況確認（閲覧のみ） |
| **アド管理** | 全タスク | 進捗更新、優先度変更、メモ |

---

## 構成案

```
adkanri-task.netlify.app（または独自URL）
├── タブ「依頼する」→ 依頼ツール（GAS iframe）
├── タブ「タスク一覧」→ task-dashboard を埋め込み or 統合
└── ログイン → 役割に応じて表示を切り替え
```

---

## 実装方針

### Phase 1: task-dashboard の拡張

1. **アド管理ルームを追加**
   - `.env` に `CHATWORK_ROOM_AD`（アド管理用ルームID）を追加
   - 依頼ツールと同じルームを使用

2. **全タスク取得モード**
   - `MY_ID` フィルタをオプション化
   - アド管理メンバー用: 全タスクを取得

3. **依頼者フィルタ**
   - タスク本文から `依頼者：xxx@domain.com` を抽出
   - 依頼者モード: 自分のメールに一致するタスクのみ表示

### Phase 2: 認証・役割

1. **ユーザー識別**
   - 案A: URLパラメータ `?email=xxx`（簡易・開発用）
   - 案B: Google OAuth（本番推奨）

2. **役割設定**
   - `.env` または `config.json` でアド管理メンバー（4人）のメール or Chatwork ID を定義
   - それ以外 = 依頼者

### Phase 3: 統合UI

1. **adkanri-task にタブ追加**
   - 「依頼する」: 現状のGAS iframe
   - 「タスク一覧」: task-dashboard を iframe で埋め込み

2. **デプロイ**
   - task-dashboard を Netlify / Vercel 等にデプロイ
   - 同一ドメイン or サブドメインで統合

---

## アド管理4人の設定例

```env
# アド管理メンバー（全体が見える人）のChatworkアカウントID
AD_MEMBER_IDS=10034061,9797164,5420288,10696465

# アド管理用ルーム（依頼ツールと同じルーム）
CHATWORK_ROOM_AD=333632829
```

---

## 実装済み（Phase 1）

- `CHATWORK_ROOM_AD`: アド管理用ルーム。設定時、`?view=all` で全タスク取得
- `?view=requestor&email=xxx@domain.com`: 自分が依頼したタスクのみ表示
- タスク本文から `依頼者：xxx@domain.com` を抽出してフィルタ

### 使い方

| URL | 表示 |
|-----|------|
| `http://localhost:3456/` | 自分のタスク（従来通り） |
| `http://localhost:3456/?view=all` | 全タスク（アド管理用・ROOM_AD を使用） |
| `http://localhost:3456/?view=requestor&email=xxx@shibuya-ad.com` | 自分が依頼したタスクのみ |

### .env の追加

```env
CHATWORK_ROOM_AD=333632829
```

※ 依頼ツールと同じルームIDを指定

---

## 次のステップ

1. adkanri-task にタブ「タスク一覧」を追加し、task-dashboard を iframe で埋め込み
2. 依頼者用: GAS からメールを渡して `?view=requestor&email=xxx` で表示
3. アド管理用: `?view=all` のURLを4人に共有
4. task-dashboard のデプロイ（Netlify / Vercel）
