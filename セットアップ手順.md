# TOアド管理依頼ツール セットアップ手順（v2）

> **まずは** `セットアップ_やることリスト.md` を開いて、チェックしながら進めてください。

## 概要

GAS Webアプリで依頼フォームを構築。**スプシ不要**。
依頼 → チャットワークタスク化 → ツール上で進捗管理。

- **AXISアドレスのみ**利用可能（メールアドレスをそのまま依頼者名に使用）

---

## セットアップ手順

### STEP 1: GASプロジェクト作成

1. 新規GASプロジェクトを作成（拡張機能 → Apps Script）
2. ファイルを作成:
   - `Code.gs` の内容を**すべて**貼り付け
   - 左の「+」→「HTML」→ ファイル名 `index` → `index.html` の内容を貼り付け

### STEP 2: Code.gs の設定

`Code.gs` 内の以下を実際の値に置換:

| 設定項目 | 説明 |
|---------|------|
| `CHATWORK_API_TOKEN` | Chatwork APIトークン |
| `CHATWORK_ROOM_ID` | 通知先ルームID |
| `ALLOWED_EMAIL_DOMAINS` | 許可するメールドメイン（例: `['axis-ads.co.jp', 'axis-hd.co.jp']`） |
| `ASSIGN_MAP` | `USER_ID_1` / `USER_ID_2` を担当者のChatworkユーザーIDに |
| `ALL_USER_IDS` | 営業時間外の通知先（カンマ区切り） |

### STEP 3: デプロイ

1. Apps Script エディタ →「デプロイ」→「新しいデプロイ」
2. 種類: 「ウェブアプリ」
3. 設定:
   - **次のユーザーとして実行: `ユーザー`**（必須）
   - アクセスできるユーザー: 組織内 or リンクを知っている全員
4. 「デプロイ」をクリック
5. 表示されるURLが依頼フォームのURL

### STEP 4: 動作確認

1. AXISアドレスでログインした状態でデプロイURLにアクセス
2. 依頼者にメールアドレスが表示されることを確認
3. 大分類 → 小分類 → 詳細入力 → 送信
4. 確認ポイント:
   - チャットワークにタスクが作成されるか
   - タスク一覧タブで進捗更新ができるか

---

## 更新時の注意

- コード修正後は「デプロイ」→「デプロイを管理」→「編集」→ バージョンを「新バージョン」にして更新
- 許可ドメインの追加・変更は `Code.gs` の `ALLOWED_EMAIL_DOMAINS` を編集
- チャットワークの通知先変更は `Code.gs` の `ASSIGN_MAP` を編集
