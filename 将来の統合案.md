# アド管理ツール統合案

依頼ツールとタスク管理ツールを合体させる場合の構成案です。

---

## 現状の対応（iframe埋め込み）

- **URL**: `adkanri-task.netlify.app` のまま（script.google.com は表示されない）
- 依頼ツール（GAS）を iframe で埋め込み
- アドレスバーには自前のURLのみ表示

---

## 将来の統合パターン

### パターンA: ラッパーでタブ統合

```
adkanri-task.netlify.app
├── タブ「依頼する」→ 依頼ツール（GAS iframe）
├── タブ「タスク一覧」→ タスク管理ツール（別システム iframe または自前UI）
└── 共通ヘッダー・ナビ
```

- Netlify で静的HTMLをホスト
- 各タブで iframe または別コンテンツを切り替え
- 依頼ツールはGASのまま、タスク管理は別サービス or 自前開発

### パターンB: フロントエンドを自前で統一

```
adkanri-task.netlify.app（React/Vue等）
├── GAS を API 化（doGet/doPost で JSON 返却）
├── フロントエンドで依頼フォーム・タスク一覧を実装
└── タスク管理機能も同じフロントに統合
```

- GAS はバックエンドAPIのみ
- フロントは Netlify / Vercel 等でホスト
- 完全に自前URLで統一可能

### パターンC: タスク管理ツールに依頼機能を組み込む

- 既存のタスク管理ツール（Notion、ClickUp、自社開発など）がメイン
- 依頼フォームをそのツール内のフォーム/ビューとして実装
- GAS は送信先・Chatwork連携のバックエンドとして利用

---

## 推奨

- **短期**: パターンA（iframe でタブ統合）が実装しやすい
- **中期**: パターンB でフロントを統一し、GAS を API 化
